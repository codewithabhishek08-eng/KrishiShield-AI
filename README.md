## 🤖 AI & Intelligence Layer

### Groq Integration Architecture

```javascript
// groqClient.js — Single source for all AI calls
const groqClient = {
  model: "llama3-70b-8192",
  cache: new Map(),           // 5-minute TTL per unique prompt
  pending: new Map(),         // Deduplication — same key = one request
  
  async call(prompt, options = {}) {
    const key = hash(prompt)
    if (this.cache.has(key)) return this.cache.get(key)
    if (this.pending.has(key)) return this.pending.get(key)
    // ... request logic
  }
}
```

## 🔐 Security

### Implementation

```
┌─────────────────────────────────────────┐
│           SECURITY LAYERS               │
├─────────────────────────────────────────┤
│                                         │
│  1. API Keys — Server-side only         │
│     GROQ_API_KEY never exposed to       │
│     client. All Groq calls routed       │
│     through /api/* Next.js routes.      │
│                                         │
│  2. Input Validation                    │
│     All user inputs sanitised before    │
│     Groq prompt injection.              │
│     Max token limits enforced.          │
│                                         │
│  3. Firebase App Check                  │
│     Prevents API abuse from             │
│     non-app clients.                    │
│                                         │
│  4. Rate Limiting                       │
│     400ms debounce on all Groq calls.   │
│     5-minute cache on identical prompts │
│     Concurrent deduplication.           │
│                                         │
│  5. Local Data Only                     │
│     Profile data stored in localStorage │
│     No PII sent to any external server  │
│     except Groq (location + crop name). │
│                                         │
└─────────────────────────────────────────┘
```
